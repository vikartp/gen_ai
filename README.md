# GenAI Projects

This repository contains multiple AI and RAG (Retrieval-Augmented Generation) projects, including Node.js and Python implementations, PDF parsing utilities, and agentic AI experiments.

## 📁 Project Structure

```
gen_ai/
├── kongsberg/              # RAG demos and training materials
│   ├── node-rag-demo/      # Node.js RAG implementation
│   ├── py-rag-demo/        # Python RAG implementation
│   └── slides/             # Training slides
├── pdf-parser/             # PDF text extraction utility
├── agentic-ai/             # Agentic AI experiments
└── README.md               # This file
```

## 🚀 Quick Start

Each subfolder has its own `README.md` with detailed setup instructions. Start by setting up the prerequisites below, then navigate to the project you're interested in.

### Prerequisites

- **Node.js** (v18 or higher) — for Node.js RAG demo
- **Python** (3.10 or higher) — for Python projects
- **Docker** (for ChromaDB server — Node.js demo only)
- **API Keys**: OpenAI or OpenRouter API key

### Clone the Repository

```bash
git clone <your-repo-url>
cd gen_ai
```

## 🔑 API Keys Configuration

RAG demos and agentic-ai support **OpenAI** and **OpenRouter** APIs:

### Using OpenAI:

```env
OPENAI_API_KEY=sk-...your-openai-key...
```

### Using OpenRouter:

```env
OPENAI_API_KEY=sk-or-v1-...your-openrouter-key...
```

The code is configured to use OpenRouter's base URL (`https://openrouter.ai/api/v1`) by default. Available model examples:

- `openai/gpt-4o-mini`
- `anthropic/claude-3-5-sonnet`
- `google/gemini-pro`

> **Security**: `.env` files (with real keys) are gitignored. Only `.env.example` (with placeholders) is tracked in Git.

## 🗂️ What Gets Regenerated

After cloning, these files/folders will be regenerated:

| Item | Generated By | Purpose |
|------|-------------|---------|
| `.env` | Copy from `.env.example` | API keys and secrets |
| `node_modules/` | `npm install` | Node.js dependencies |
| `.venv/` | `python -m venv` or `uv venv` | Python virtual environments |
| `chroma_data/` | `build-index.mjs` | Node.js vector database |
| `chroma-store/` | `build_index.py` | Python vector database |
| `__pycache__/` | Python runtime | Python bytecode cache |

## 📝 Notes

- Vector databases (`chroma_data/`, `chroma-store/`) are NOT committed — must be regenerated
- `.env` files with real API keys are NOT committed
- `.env.example` templates ARE committed
- Generated text files (`*_extracted.txt`) are NOT committed
- Lock files (`uv.lock`, `package-lock.json`) may or may not be committed depending on team preference
