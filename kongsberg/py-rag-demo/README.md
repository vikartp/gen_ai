# Python RAG Demo

A Retrieval-Augmented Generation (RAG) implementation using **Chroma** (persistent storage, no Docker needed), **LangChain**, and **OpenAI/OpenRouter** APIs with LCEL (LangChain Expression Language).

## Prerequisites

- **Python** 3.10 or higher
- **uv** (recommended) or pip
- **API Key**: OpenAI or OpenRouter

## Setup

### 1. Create Virtual Environment

```bash
python -m uv venv
```

### 2. Activate Virtual Environment

- **Windows:** `.venv\Scripts\activate`
- **Linux/Mac:** `source .venv/bin/activate`

### 3. Install UV (if not already installed)

```bash
pip install uv
```

**Alternative (Windows — proper install):**

```powershell
# Uninstall pip version if present
pip uninstall uv -y

# Install properly
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
# Follow terminal guide after this
```

### 4. Install Dependencies

```bash
python -m uv pip install langchain-openai langchain-chroma langchain langchain-text-splitters python-dotenv --python .venv\Scripts\python.exe
```

Or use the automated setup script from the `kongsberg/` directory:

```bash
cd ..
bash rag-setup-py.sh
```

### 5. Configure Environment Variables

```bash
cp .env.example .env
```

Edit `.env` and replace `your-api-key-here` with your actual API key.

**Windows (PowerShell):**

```powershell
Copy-Item .env.example .env
```

> You can use either OpenAI (`sk-...`) or OpenRouter (`sk-or-v1-...`) API keys. The project is configured to use OpenRouter's base URL by default.

### 6. Build the Vector Index

```bash
python build_index.py
```

### 7. Query the RAG System

```bash
python query_rag.py
```

## Key Files

| File | Description |
|------|-------------|
| `build_index.py` | Creates vector embeddings and stores them in Chroma |
| `query_rag.py` | Queries the RAG system using LCEL |
| `.env.example` | Environment variable template |

## Model Configuration

You can modify model names in `build_index.py` and `query_rag.py`. Available OpenRouter models:

- `openai/gpt-4o-mini`
- `anthropic/claude-3-5-sonnet`
- `google/gemini-pro`

## Troubleshooting

### ModuleNotFoundError

Ensure virtual environment is activated and dependencies are installed:

```bash
.venv\Scripts\activate
python -m uv pip install <missing-package>
```

### ChromaDB Python server not running

```bash
pip install chromadb
python -m chromadb.cli.cli run --host localhost --port 8000
```

### OpenAI API rate limits

Consider using OpenRouter for access to multiple LLM providers with better rate limits.
