# Node.js RAG Demo

A Retrieval-Augmented Generation (RAG) implementation using **ChromaDB** (via Docker), **LangChain**, and **OpenAI/OpenRouter** APIs.

## Prerequisites

- **Node.js** v18 or higher
- **Docker** (for ChromaDB server)
- **API Key**: OpenAI or OpenRouter

## Setup

### 1. Install Dependencies

```bash
npm install
```

### 2. Configure Environment Variables

```bash
cp .env.example .env
```

Edit `.env` and replace `your-api-key-here` with your actual API key.

**Windows (PowerShell):**

```powershell
Copy-Item .env.example .env
```

> You can use either OpenAI (`sk-...`) or OpenRouter (`sk-or-v1-...`) API keys. The project is configured to use OpenRouter's base URL by default.

### 3. Start ChromaDB Server (Docker)

```bash
docker run -d -p 8000:8000 chromadb/chroma
```

Or use the automated setup script from the `kongsberg/` directory:

```bash
cd ..
bash rag-setup-js.sh
```

### 4. Build the Vector Index

```bash
node build-index.mjs
```

Or using npm script:

```bash
npm run build:index
```

### 5. Query the RAG System

```bash
node query-rag.mjs
```

Or using npm script:

```bash
npm run query:rag
```

## Key Files

| File | Description |
|------|-------------|
| `build-index.mjs` | Creates vector embeddings and stores them in ChromaDB |
| `query-rag.mjs` | Queries the RAG system |
| `package.json` | Node.js dependencies |
| `.env.example` | Environment variable template |

## Dependencies

- `langchain` / `@langchain/openai` / `@langchain/core` — LangChain framework
- `chromadb` — ChromaDB vector database client
- `dotenv` — Environment variable management

## Model Configuration

You can modify model names in `build-index.mjs` and `query-rag.mjs`. Available OpenRouter models:

- `openai/gpt-4o-mini`
- `anthropic/claude-3-5-sonnet`
- `google/gemini-pro`

## Troubleshooting

### ChromaDB connection error

Ensure the Docker container is running:

```bash
docker run -d -p 8000:8000 chromadb/chroma
```

### OpenAI API rate limits

Consider using OpenRouter for access to multiple LLM providers with better rate limits.
